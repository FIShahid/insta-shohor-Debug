Question and Answer

JavaScript is a widely popular Client side scripting language. Which means it runs on the client machines web browser with the help of browsers built in JS engine. For example Google Chrome has V8 engine ,Mozila has Spider Monkey and so on. What these Engine does is when we try to run the JS code in the browser it parses the code, it means it checks the code line by line from the top and then read it through bottom. And if everything checks out that is no error is found it then converts it computer readable Machine code and then runs the code and we can see our desired output.

We know that JavaScript has a combination of both synchronous and asynchronous events. synchronous means it executes statement line by line whereas asynchronous is call backs and promises. JS is single threaded. That means it does one thing at a time. Event loop is responsible for executing the codes that means checking call stacks ,collecting and processing events and executing queued sub-tasks. If all the statement from callback finish execution event loop will pop the statement from event queue and push it to call stack.

Local storage and Session storage are both client-side storage. Most of their methods are also the same However there is a slight difference. As the name applies the session storage is active until the browser session is active which means when we open a page or tab in the browser it stores the session data and we can access them but when we close the tab or page the session expires so does the storage. But local storage data doesn't expire until it was manually cleared and can be accessed anytime. And only the client and read both storage data.
